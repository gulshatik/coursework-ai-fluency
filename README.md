# ai-fluency

## План личной AI‑грамотности (Personal AI Fluency Plan)

> **Цель** – систематизировать знания и навыки в области искусственного интеллекта, чтобы стать уверенным пользователем ИИ‑технологий и способным применять их в профессиональной деятельности.

---

### 1. Личные цели и мотивация

| Цель | Почему это важно для меня | Как измерить прогресс |
|------|--------------------------|-----------------------|
| **Понимание принципов работы моделей** | Позволит критически оценивать результаты ИИ и выбирать подходящие инструменты | Успешное объяснение архитектуры GPT‑4 в собеседовании |
| **Разработка собственных промптов** | Способность быстро получать нужные ответы от LLMs ускоряет работу | 5 успешных промптов, решающих реальные задачи (см. раздел 4) |
| **Этическое использование ИИ** | Предотвращение вреда и соблюдение нормативов | Соблюдение всех пунктов кодекса этики в проектах |
| **Непрерывное обучение** | Технологии быстро меняются, нужно быть в курсе новейших исследований | 3 новых статьи/курса в месяц |

---

### 2. Основные области навыков

| Область | Ключевые темы | Примерный объём изучения |
|---------|---------------|--------------------------|
| **1️⃣ Данные и подготовка** | Сбор, очистка, аугментация данных; метрики качества | 20 ч |
| **2️⃣ Архитектуры моделей** | Transformer, RNN, CNN, Diffusion | 30 ч |
| **3️⃣ Prompt Engineering** | Структурирование запросов, контекстуализация, chain‑of‑thought | 25 ч |
| **4️⃣ Этические и правовые аспекты** | Bias, privacy, explainability, GDPR | 15 ч |
| **5️⃣ Инструменты и экосистема** | OpenAI API, Hugging Face, LangChain, LlamaIndex | 20 ч |

---

### 3. Путь обучения

| Область | Ресурс | Формат | Оценка времени |
|---------|--------|--------|----------------|
| **Данные и подготовка** | *Data Science Handbook* (O’Reilly) | Книга | 10 ч |
| | Kaggle “Intro to Data Cleaning” | Онлайн‑курс | 5 ч |
| **Модели** | Coursera – “Deep Learning Specialization” (Andrew Ng) | Серия курсов | 15 ч |
| | Fast.ai – “Practical Deep Learning for Coders” | Видео + практики | 10 ч |
| **Prompt Engineering** | OpenAI Cookbook | Документация + примеры | 8 ч |
| | Prompt Engineering Handbook (GitHub) | Чтение и эксперименты | 7 ч |
| **Этика** | “Ethics of AI” – edX | MOOC | 6 ч |
| | “AI Ethics: A Guide for the Responsible Use of Artificial Intelligence” (Harvard) | Книга | 9 ч |
| **Инструменты** | LangChain Docs | Текст + примеры кода | 5 ч |
| | Hugging Face Spaces – “🤗 Transformers” | Практика | 10 ч |

> **Итого:** ~200 часов (≈8 недель при 25 ч/нед.)

### 4. Практические проекты

| Проект | Цель | Ожидаемый результат | Срок |
|--------|------|---------------------|------|
| **1️⃣ Чат‑бот для FAQ** | Применить prompt engineering и LangChain | Бот, отвечающий на вопросы компании | 2 недели |
| **2️⃣ Анализ тональности отзывов** | Обучить модель классификации | Отчёт с метриками точности | 3 недели |
| **3️⃣ Генерация кода по описанию** | Использовать LLM для автогенерации функций | Пакет Python‑скриптов | 4 недели |
| **4️⃣ Этический аудит модели** | Проверить bias и fairness | Доклад с рекомендациями | 2 недели |

> Каждый проект сопровождается GitHub репозиторием, README и Jupyter Notebook.

### 6. Оценка и обратная связь

| Метрика | Как измерять | Целевой показатель |
|---------|--------------|---------------------|
| **Точность моделей** | Accuracy, F1‑score | ≥ 85 % |
| **Качество промптов** | Количество корректных ответов без уточнений | ≥ 90 % |
| **Этическая оценка** | Отсутствие обнаруженных bias | 0/10 |
| **Соблюдение сроков** | Проекты завершены в планируемый период | 100 % |

> Регулярные ревью каждые 2 недели: анализ прогресса, корректировка плана.
